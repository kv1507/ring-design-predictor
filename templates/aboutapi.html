API & Backend Documentation

Detailed explanation of the Flask API and backend functionalities.

Project Overview

This API is built using Flask and serves as the backend for the Ring Design Choice Prediction application. It loads a trained model to predict whether a jeweler's party will select a particular ring design.

Technologies Used

  • Flask (Python web framework)
  • Pandas (Data manipulation)
  • Pickle (Model persistence)
  • Scikit-learn (Machine learning utilities)

Model & Preprocessing

The API loads the trained model, label encoder, feature names, and scaler using Pickle. Data preprocessing includes:

  • One-hot encoding categorical features
  • Handling missing values
  • Feature scaling using MinMaxScaler (if available)
  • Ensuring feature alignment with training data

API Endpoints

1. Home Page

URL: /

Method: GET

Description: Serves the frontend HTML form.

2. Predict Ring Design Choice

URL: /predict

Method: POST

Description: Predicts if a jeweler's party will select a specific ring design.

Request JSON Format:

{
    "Party Name": "ABC Jewels",
    "Number of Diamonds": 5,
    "First Color of Diamond": "White",
    "Second Color of Diamond": "Blue",
    "Third Color of Diamond": "None",
    "Shape": "Round",
    "Quantity": 10,
    "type": "Luxury"
}

Response JSON Format:

{
    "prediction": "Yes"
}

Error Handling

The API includes error handling for:

  • Missing required fields
  • Invalid quantity values
  • Processing errors

Conclusion

The API provides a structured approach to predicting ring design choices using machine learning. Proper preprocessing and error handling enhance model accuracy and reliability.
